<threat_model>
Trust boundaries:
- Current local workspace versus future Cloudflare MCP production runtime.
- Local generated evidence files versus production audit records.
- Development/migration public surfaces versus production public endpoint.
- File-hash evidence versus Git commit evidence.

STRIDE register:
- TM-01 Spoofing: GPTs/ngrok/OpenAPI docs can look like production public interfaces. Mitigation: mark these files as development or migration references only.
- TM-02 Tampering: Generated artifacts can be overwritten by tests and mistaken for source. Mitigation: add `.gitignore` and source hygiene policy.
- TM-03 Repudiation: Local audit CSV/NDJSON lacks production actor identity. Mitigation: document audit boundary and avoid production audit claims.
- TM-04 Information disclosure: Generated OpenAPI/ngrok docs may contain stale public URLs or demo credentials. Mitigation: add dev-only markers and later secret scan gate.
- TM-05 Denial of service: Live-service checks may fail because local ports are unavailable. Mitigation: report live-service checks separately from unit and contract checks.
- TM-06 Elevation of privilege: Write/upload/deploy endpoints could be treated as safe production behavior. Mitigation: keep V1 read-only and defer write tools.
</threat_model>
