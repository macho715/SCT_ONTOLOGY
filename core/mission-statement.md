# sct_ontology Mission Statement

Status: Approved operating governance v1.0
Owner: MR.CHA / HVDC Logistics
Source: `sct_ontology_MCP_operating_update_v1.0.md`

## Mission

`sct_ontology` is the team-standard LLM operating layer for HVDC project logistics.

Its mission is to:

- reduce hallucination in ChatGPT and other LLM answers;
- inject MR.CHA's HVDC logistics domain knowledge, terms, workflow, evidence rules, and decision gates into the answer process;
- give MR.CHA and team members the same operating context, vocabulary, evidence requirements, and risk judgment when using LLM tools.

## Operating Principles

1. Treat HVDC logistics context as the default operating context.
2. Answer from approved evidence before making factual logistics claims.
3. Keep high-risk business decisions behind AMBER/ZERO gates.
4. Preserve human approval for close, write-back, external send, report publication, cost approval, and transaction mutation.
5. Validate team consistency through Golden Q&A regression and GitHub governance.

## Boundaries

This mission statement is governance evidence.
It does not create a new runtime MCP tool, production write-back path, or live KG implementation.
